# 证证鸽用户操作流程文档

> 文档类型：用户操作与测试基线  
> 版本：v0.1  
> 更新时间：2026-04-11  
> 说明：本文件从测试和联调角度拆出用户操作流程。后续做页面联调、手工测试、录制 Demo、编写测试用例，都应以此文档为操作基准。

## 1. 使用说明

这份文档不讨论技术实现，只讨论：

- 用户从哪里进入
- 点什么
- 系统应该返回什么
- 哪些地方需要人工确认

## 2. 当前支持的核心流程

当前版本已经具备雏形或最小可验证链路的流程：

1. 主站查看产品资料
2. 进入工作台查看案件
3. 浏览器插件发起一键取证
4. 服务端创建案件与证据包
5. 工作台查看案件详情和证据概览
6. 查看模板列表

## 3. 当前尚未完整支持的流程

这些流程已经在 PRD 中定义，但当前还没有完全做完：

1. 证据包详情页
2. 文书草稿生成
3. 文书审核与导出
4. 监控任务创建与执行
5. 通知配置
6. 登录和角色权限

## 4. 用户流程一：主站到工作台

### 目标

用户先理解产品，再进入工作台。

### 标准步骤

1. 打开主站首页 `/`
2. 查看：
   - 产品概览
   - 核心能力
   - 功能链路
   - 产品资料
3. 点击 `进入工作台`
4. 进入 `/workspace`

### 期望结果

- 页面能正常打开
- 不出现内部实现词
- 能跳转到工作台总览

### 检查点

- 不应出现：钉钉、邮箱、权限模型、Hermes、mock、API 已配置等内部词

## 5. 用户流程二：查看产品资料

### 目标

用户查看对外资料页，理解产品定位和边界。

### 标准步骤

1. 进入 `/docs`
2. 依次查看：
   - 产品概览
   - 适用场景
   - 能力边界
3. 点击 `进入工作台`

### 期望结果

- `/docs` 只展示对外资料
- 不暴露内部 PRD、技术选型、源码路径

## 6. 用户流程三：插件一键取证

### 目标

用户在疑似侵权页面发起取证。

### 标准步骤

1. 打开一个公开网页
2. 打开浏览器插件 popup
3. 点击 `一键取证并提交`
4. 等待状态变更
5. 如有工作台链接，点击打开

### 期望结果

- 插件能展示当前页面标题、URL、采集时间
- 插件能返回取证编号
- 插件不应暴露后端实现细节

### 当前已实现

- 页面信息采集
- 状态提示
- 结果跳转工作台

### 当前未实现

- 真正的 LLM 分析
- 真正的通知发送
- 真正的文书生成

## 7. 用户流程四：工作台查看案件列表

### 目标

用户进入工作台后查看案件总览。

### 标准步骤

1. 打开 `/workspace/cases`
2. 查看案件卡片：
   - 标题
   - 风险状态
   - 来源
   - 更新时间
3. 点击任意案件进入详情页

### 期望结果

- 案件列表能展示
- 状态和来源用业务语言展示
- 不展示 `后端 API / mock` 之类词

## 8. 用户流程五：工作台查看案件详情

### 目标

用户查看案件详情、证据概览和下一步建议。

### 标准步骤

1. 打开 `/workspace/cases/{caseId}`
2. 查看：
   - 案件摘要
   - 风险评分
   - 证据包概览
   - 处理建议
   - 基础证据目录
3. 点击 `去模板页`

### 期望结果

- 能看到风险评分和证据概览
- 有“疑似侵权 / 建议人工复核”边界提示
- 不直接给出“已侵权”结论

## 9. 用户流程六：模板查看

### 目标

用户查看当前可用模板。

### 标准步骤

1. 打开 `/workspace/templates`
2. 查看模板卡片：
   - 律师函
   - 平台投诉函
   - 举报材料草稿

### 期望结果

- 页面能正常展示
- 模板说明是产品语言，不是开发占位语言
