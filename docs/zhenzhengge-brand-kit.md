# 证证鸽品牌方案与展示口径

> 本文档用于统一项目在海报、路演、答辩、工作台、插件和机器人中的品牌表达，保证“可爱、有记忆点、但不失专业”。

## 1. 主品牌名

### 最终推荐

`证证鸽`

### 命名理由

- 与“证据、取证、固证、举证”天然关联
- 读起来轻巧，有 IP 感，适合比赛传播
- 不会像“公平鸽”那样过于偏向司法裁判语境
- 更贴近当前产品重心：证据先行、处置跟进

## 6. IP 形象设定

### 角色名称

`证证鸽`

### 角色定位

一只负责巡查、留证、送达提醒的智能鸽子，不负责裁判谁对谁错，而是负责把证据稳稳送到位，把案件材料整理清楚，把后续动作衔接起来。

### 角色关键词

- 巡查
- 送达
- 守护
- 留证
- 提醒
- 可靠

### 角色人设

- 不做“法官”
- 不扮“律师”
- 更像“证据快递员 + 风险提醒助手 + 法务小秘书”

### 视觉建议

- 主色：蓝、白、浅灰
- 辅色：浅金或浅青，增加识别度
- 元素：放大镜、证据包、信封、盾牌、时间戳、网页快照边框
- 风格：亲和、克制、可信，不做儿童化夸张设计

## 7. 模块命名建议

为了让整个产品既有 IP 体系，也方便海报和答辩展示，建议采用“鸽 + 功能”的命名结构。

### 主模块命名

- `鸽眼`：监测与识别
- `鸽证`：取证与固证
- `鸽信`：消息推送
- `鸽函`：文书生成
- `鸽审`：人工审核
- `鸽库`：案件与证据归档

### 模块说明

#### 鸽眼

负责：

- 主动比对
- 自动巡检
- 相似度计算
- 风险评级

#### 鸽证

负责：

- 页面截图
- HTML 保存
- 文本提取
- 哈希计算
- 时间留痕
- 证据包生成

#### 鸽信

负责：

- 邮件推送
- 钉钉机器人通知
- 风险摘要卡片
- 后续可扩展飞书、企业微信、短信

#### 鸽函

负责：

- 律师函初稿
- 平台投诉函
- 举报材料草稿
- 侵权分析说明
- 证据目录

#### 鸽审

负责：

- 人工审核
- 风险口径修正
- 模板修订
- 导出确认

#### 鸽库

负责：

- 案件归档
- 权利资产关联
- 历史记录
- 证据包管理
- 审计日志

## 8. 面向不同场景的口径

### 面向评委

> 证证鸽不是简单的侵权监测工具，而是一条从线索发现、证据固化到法务材料生成的侵权响应闭环。

### 面向潜在用户

> 发现侵权后，不再只是手动截图。证证鸽帮你把网页内容快速沉淀为标准化证据包，并进入后续法务处置流程。

### 面向技术评审

> 我们采用“插件 + 工作台 + 后台 Agent”的架构，把前端一键取证、服务端证据归档、风险分析与文书生成编排连接起来。

## 9. 答辩开场白建议

### 30 秒版

> 大家好，我们的项目叫证证鸽。它聚焦知识产权侵权处理最容易被忽略、但最关键的前置环节：取证固证。我们希望把分散、易失的网页侵权线索，快速转化为标准化证据包、风险事件和法务材料初稿，帮助用户更高效地推进后续投诉、发函和维权准备。

### 60 秒版

> 大家好，我们带来的项目是证证鸽，一套面向知识产权侵权响应场景的智能取证与处置辅助平台。当前很多品牌方、创作者和法务团队在面对侵权时，第一步往往只能靠手动截图，证据零散、容易丢失，也难以直接进入后续法务流程。证证鸽提供两种模式：一是用户在浏览器中发现疑似侵权页面后，一键取证、自动固证；二是平台后台持续巡检指定站点，发现高风险线索后自动留痕并推送通知。在此基础上，我们再结合模板化文书生成和人工审核机制，把侵权线索真正转化为可行动的案件材料。
