# 证证鸽｜AI 知识产权侵权响应平台

## 让疑似侵权页面，更快变成可审核、可导出的案件材料与文书草稿

---

## 一句话定位

**证证鸽 = 取证留痕 + 案件沉淀 + AI 草稿生成**

从发现侵权线索到沉淀证据包、生成案件材料与文书草稿，证证鸽打通了一条可执行的自动化链路，让法务团队和品牌方能够更高效地响应知识产权侵权问题。

---

## 两大用户操作链路（业务闭环）

### 链路一：主动取证 → 一键生成草稿

```
发现疑似侵权页面
      ↓
点击浏览器插件「一键取证」
      ↓
系统自动采集：URL、标题、正文、HTML、截图、抓取时间
      ↓
系统自动创建「案件」+「证据包」
      ↓
【AI 自动生成】法律文书草稿（律师函 / 平台投诉函）
      ↓
人工审核 → 导出并进入后续处置
```

**适用场景**：法务人员主动巡查、人工发现疑似侵权页面

---

### 链路二：智能监控 → 自动生成草稿

```
配置监控任务：目标站点 + 关键词 + 巡检频率 + 风险阈值
      ↓
系统自动巡检，持续发现高风险页面
      ↓
命中规则 → 自动创建「案件」+「证据包」
      ↓
【AI 自动生成】法律文书草稿（律师函 / 平台投诉函）
      ↓
人工审核 → 导出并进入后续处置
```

**适用场景**：品牌方持续监测竞品侵权、电商平台监测假冒店铺

---

## AI 在其中发挥了什么价值？

### 1. Hermes Agent — 任务编排中枢

Hermes 是证证鸽的自动化引擎，负责把取证、研判、通知、文书生成串成连续流程：

| 传统方式 | 使用 Hermes 后 |
|---------|--------------|
| 人工手动收集截图 | 插件一键提交，后台自动抓取 |
| 人工整理证据目录 | 系统自动生成证据包清单 |
| 人工撰写文书 | AI 参照模板自动生成结构化草稿与 Word 初稿 |
| 多人多次传递 | 全流程自动衔接，律师聚焦审核与处置决策 |

**Hermes 的核心价值**：把"人找事"变成"事找人"，让团队专注于决策而非执行。

---

### 2. AI 文书生成 — 标准化草稿与 Word 导出

证证鸽当前已内置模板类型与导出结构，AI 参照既有法律文书模板的语言风格和格式规范，将证据内容组织为结构化草稿并导出：

- **律师函**：结构清晰、便于律师继续审核的侵权处置初稿
- **平台投诉函**：符合平台投诉场景的规范化材料草稿
- **证据目录**：结构化的证据清单，便于归档、补强和审核

当前输出包括结构化草稿和可导出的 `.docx` 文档。其定位是**可审核、可继续完善、可继续导出**的法律文书草稿，而不是系统直接替代律师形成最终法律结论。

## 关于取证渠道的说明

当前版本采用 Playwright 技术进行页面抓取、页面截图与代码留存。

> **由于技术及国家认证要求等因素，暂时还未接入可信时间戳的 API 渠道。**

网页截图、HTML 留存与页面正文摘录，在当前阶段主要用于内部研判、证据沉淀和草稿生成。如需进入正式司法或高强度对外维权流程，仍建议后续补充可信时间戳等更强的固证材料。

---

## 后续规划：可信时间戳固证

正式投产后，有望在**「取证固证」环节**接入可信时间戳功能，实现：

- 取证时间即时固化
- 哈希值自动计算与存证
- 与主流时间戳机构 API 对接

届时，证证鸽将形成**「发现线索 → 取证留痕 → 固证补强 → 草稿生成」**的完整闭环，进一步提升取证固证的精准度与可追溯性。当前项目和程序逻辑，也会持续朝这个方向预留和靠拢。

## 总结

证证鸽解决的是一个具体且高频的问题：

**把「网页侵权线索」，变成「可继续审核、可继续完善、可继续导出的案件材料与法律文书草稿」**

围绕这个目标，当前版本已经完成了从取证固证到文书生成的完整链路。AI 在其中承担了两大核心角色：

1. **Hermes Agent**：把人工串联的多步骤流程，编排为自动连续的任务流
2. **AI 文书生成**：把证据内容自动填入专业模板，生成结构化草稿与可导出的 Word 初稿

最终交付给律师的，不再只是散乱的截图和链接，而是一套**证据包 + 案件材料 + 可继续审核的 Word 草稿**。
