# 证证鸽比赛冲刺清单（2026-04-20）

## 1. 已确认跑通（可用于路演）

1. 插件 intake -> 案件 -> 证据包 -> 草稿 -> 可编辑保存 -> docx 导出已实测通过。
2. 前后端 + 插件均可启动：`pnpm dev:web`、`pnpm dev:extension`、`uv ... uvicorn`。
3. 鉴权与角色已落地（`viewer/operator/admin`），并接入 Web 代理透传。
4. 审计日志已落地（intake、draft create/update/approve/export 等关键动作）。
5. 监控任务已支持调度器与执行历史。
6. Harness/Hermes 已接入并可启用（`engine=harness_cli`）。
7. 文书导出为可编辑 `.docx`（非只读 PDF/截图）。
8. 组织级 ACL 已接入（token 携带 `user_id/role/org_id`，接口按组织作用域过滤）。
9. 可信时间戳已接入（RFC3161 TSA，可配置启用；证据包支持 `.tsr` 回执下载）。
10. 草稿编辑器已升级为富文本编辑（段落/标题/样式/列表/引用/撤销重做）。

## 2. 仍未完成 / 半完成 / 有风险的能力

### 2.1 仍未完成（必须明确）

1. 用户体系未落地：无注册/登录/组织成员管理，仅 token 级鉴权。
2. 细粒度资源权限仍待增强：当前已落地组织级 ACL，但案件级共享/成员级 ACL 仍需补充。
3. 可信时间戳已接入但默认关闭：未配置 TSA 时只会保留状态与告警，不会自动签发。
4. 平台化投诉自动提交未落地：当前是文书导出，不是对接各平台表单 API。
5. 电子签章/法证服务未落地：无法主张“司法级存证”能力。

### 2.2 半完成（可演示但不宜过度承诺）

1. 风险识别：当前监控评分是规则/文本相似度启发式，不是成熟多模态侵权判定。
2. 文书生成：已接 Harness/LLM，但法务规则库深度仍不足，仍需人工审核。
3. 通知能力：SMTP/钉钉链路已接入，但需真实账号联调后才能作为“已商用”承诺。
4. 编辑器：已升级富文本编辑，但仍非多人协作级 WPS 完整能力。

### 2.3 可能被评委质疑“像 demo”的点（需主动降风险）

1. 插件保留了模拟提交开关（默认关闭）：路演必须展示 `ALLOW_SIMULATED_SUBMISSION=false`。
2. 证据抓取存在 HTTP 回退：需在演示中说明“回退但不伪造内容”。
3. LLM 异常回退历史：已新增 `ZHZG_DRAFT_GENERATION_STRICT=true`，路演环境必须开启。

## 3. 比赛差异点（建议统一外宣）

唯一主轴：

> 不是“只发现风险”，而是把线索直接变成可执行的法务动作。

建议固定表达：

1. `线索 -> 证据包 -> 案件 -> 文书 -> 导出` 是闭环，不是单点检测。
2. 交付物是法务可继续推进的材料（可审核、可编辑、可导出）。
3. 插件 + 工作台 + 后端编排是产品形态，不是模型 API 拼接 Demo。

## 4. 合规必做（比赛“至少达标”）

1. 全程关闭模拟模式并现场验证：
   - `PLASMO_PUBLIC_ALLOW_SIMULATED_SUBMISSION=false`
   - `ZHZG_DRAFT_GENERATION_STRICT=true`
2. 路演文案禁用绝对化表述：
   - 禁说“自动判定侵权成立”
   - 改为“疑似风险识别 + 人工审核决策”
3. 明确证据边界：
   - 说明当前为网页抓取留痕与哈希链
   - 说明可信时间戳为后续接入项
4. 导出物可复核：
   - 文书主张关联 EvidenceID
   - 证据包可下载 HTML/截图
5. 提交材料一致性：
   - 海报、视频、文档中的功能描述必须与实际页面一致
