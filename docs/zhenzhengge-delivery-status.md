## 1. 当前结论

证证鸽已经从“纯文档阶段”进入“可运行骨架 + 已打通核心联调链路阶段”。

- 主站 Landing Page
- 工作台基础壳子
- 浏览器插件基础取证能力
- FastAPI 后端基础接口

- 插件 -> 后端 -> 工作台 的最小链路
- 草稿、监控、接收方式的真实 API 与页面

- 自动监测任务完整调度与定时执行

## 3. 已验证链路

已实测通过的链路：

1. 后端健康检查通过
2. Web lint / build 通过
3. 插件 build 通过
4. API pytest / smoke test 通过
5. 手工向 `/api/v1/evidence/intake` 提交插件契约 payload 可成功创建案件和证据包，并返回 `case` / `evidence_pack`
6. 工作台 `/workspace/cases` 已切换到真实 API 数据源
7. 工作台 `/workspace/cases/[caseId]` 可显示真实案件详情与证据包
8. 工作台 `/workspace/drafts` 可生成草稿并跳转详情页
9. 工作台 `/workspace/drafts/[draftId]` 可导出草稿
10. 工作台 `/workspace/monitoring` 可执行监控任务，并在命中后创建案件和证据包
11. 工作台 `/workspace/notifications` 可发送测试消息并写入通知日志
12. 工作台 `/workspace/notifications` 可查看最近通知日志
13. Landing Page、文档入口、工作台入口已通过浏览器验收
14. `next start` 生产态链路已验证可用
15. 证据包详情页已支持截图预览、HTML 片段预览和下载入口

## 4. 当前缺口

### P0 缺口

- 登录 / 权限 / 审计仍未落地
- 自动监测还没有定时调度器，当前以手动执行为主
- 文书导出仍以 Markdown 为主，未完成 docx/pdf 正式导出

### P1 缺口

- 邮件真实 SMTP 联调
- 钉钉真实 webhook 联调
- 文书模板真实渲染与 docx/pdf 导出
- 审核意见与导出历史的更完整展示

### P2 缺口

- Hermes 完整编排接入
- 历史案例驱动优化
- 飞书 / 企业微信扩展
- 海外站点扩展

## 5. 当前风险

1. 插件与 intake 字段契约已修复，但后续仍需要继续约束返回结构和错误态展示。
2. 当前 API 虽已接 SQLite 与多模块接口，但仍偏 demo 形态，尚未进入正式业务模型。
3. 插件当前是 popup 模式，尚未进入更强的 content script / background 体系。

## 6. 下一步优先级

1. 登录 / 权限 / 审计
2. 自动监测定时调度与任务历史
3. docx/pdf 导出与模板渲染
4. 通知日志与审核历史联动
5. 接口 / 数据库 /任务拆分继续跟文档基线对齐
